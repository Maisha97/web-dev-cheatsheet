1. HTML: Structure, Tables & Semantics

Building Blocks

  • <!DOCTYPE html>: Tells the browser this is an HTML5 document.
  • <html>: The root element of the page.

Semantic Elements

Using tags that describe their meaning (SEO & Accessibility):

<header>, <nav>, <main>, <section>, <article>, <aside>, <footer>

HTML Tables

<table>
  <tr> <!-- Table Row -->
    <th>Header</th> <!-- Table Heading -->
  </tr>
  <tr>
    <td>Data</td> <!-- Table Data/Cell -->
  </tr>
</table>

2. CSS: Visuals, Selectors & Positioning

Selectors & Visual Rules

  • Universal: * { box-sizing: border-box; }
  • Class vs ID: Use .className for multiple elements; #idName for one unique element.
  • Specificity: Inline style > ID > Class > Type.

Display & Positioning

/* Display */
display: block;   /* Takes full width */
display: inline;  /* Only takes content width */
display: none;    /* Removes from page */

/* Positioning */
position: relative; /* Stays in flow, can move slightly */
position: absolute; /* Moves relative to nearest parent */
position: fixed;    /* Stays on screen during scroll */

3. Developing Locally & The Command Line

Using the Terminal/CLI to manage your project files efficiently.

ls      /* List: Show all files in current folder */
cd ..   /* Change Directory: Move up one folder */
mkdir   /* Make Directory: Create a new folder */
touch   /* Create a new file (e.g., touch index.html) */
pwd     /* Print Working Directory: Where am I? */

Local Workflow Tips

  • Live Server: Use a VS Code extension to see changes instantly.
  • File Structure: Always keep your index.html in the root folder.
  • Browser Tools: Press F12 or Cmd+Opt+I to debug CSS.

4. Deployment Checklist

Follow these steps to take your site live:

  1. Validate: Check your HTML/CSS for unclosed tags.
  2. Relative Paths: Ensure images use src="./img/photo.jpg".
  3. GitHub: Initialize a repository and push your code.
  4. GitHub Pages: Go to Settings > Pages and select the 'main' branch.

Pro-Tip: Always test your live link on a mobile phone to check responsiveness!
